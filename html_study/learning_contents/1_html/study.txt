HTML(HyperText Markup Language)이란?
    HyperText : 링크를 통해 문서 간 이동이 가능한 텍스트 - 사용자가 클릭하면 다른 문서나 위치로 이동할 수 있게 하는 링크 기능을 가진 텍스트
    Markup Language : 문서의 구조와 표현 방식을 정의하는 언어

    웹 문서의 구조를 정의하는 기본 언어
        즉, HTML은 웹 문서의 뼈대를 만드는 언어

HTML4 vs HTML5
    HTML4
        구조 : <div> 중심 - 문서 구조가 명확하지 않아 가독성과 유지보수 측면에서 불편함이 있었음
        멀티미디어 : 오디오나 비디오를 삽입하려면 Flash 같은 외부 플러그인이 필요했음
        폼 기능 : HTML4는 폼(form) 요소에서 제공하는 기능이 제한적
        사용자 데이터 : 사용자 정의 데이터를 저장할 수 있는 방법이 명확하지 않았음
        접근성 : HTML4는 접근성을 위한 공식적인 지원이 부족

    HTML5
        구조 : <header>, <footer>, <section>, <article> 등 시멘틱 태그를 도입하여 문서의 구조를 의미론적으로 표현할 수 있도록 개선
        멀티미디어 : <audio>와 <video> 태그를 통해 브라우저 자체에서 멀티미디어를 직접 지원
        폼 기능 : 유효성 검사 기능이 내장되 있고, email, date, range 등 다양한 입력 다입이 추가되어 폼 기능 강화
        사용자 데이터 : data-* 속성을 통해 사용자 정의 데이터를 태그에 저장할 수 있게 되어 자바스크립트와의 연동이 용이해졌음
        접근성 : ARIA(Accessible Rich Internet Applications) 속성을 공식적으로 지원하여 장애인 접근성 향상에 기여함

-------------------------------------------------------------------------------------------------------------------
HTML로 문서 구조화는 2가지 관점으로 봐야 함
    1. 웹 브라우저 관점 : 브라우저가 HTML을 렌더링하고, CSS(디자인)와 JavaScript(동작)를 통해 요소를 선택하고 조작하기 쉽게 하기 위해서
        시멘틱 태그를 사용하여 구조가 명확해져서:
        - CSS에서 정확한 선택자가 사용 가능해짐
        - JavaScript에서도 요소 접근이 쉬워짐

    2. SEO(검색 최적화) 관점 : 검색 엔진의 크롤러가 HTML 구조를 보고, 어떤 부분이 중요한 컨텐츠인지 의미를 이해할 수 있도록 하기 위해서
        시멘틱 태그를 사용하여 구조가 명확해져서:
        - 검색 엔진(크롤러)은 그 영역이 메뉴인지, 본문인지, 광고인지 더 잘 파악함
        - 따라서 검색 결과에서 더 높은 평가를 받을 가능성이 높아짐

    정리
    - 웹 브라우저 관점 : "프론트엔드 개발의 편의성"을 위해 구조화
    - SEO 관점 : "검색 엔진(크롤러)이 의미를 이해하도록" 구조화

-------------------------------------------------------------------------------------------------------------------
웹 브라우저와 HTML의 관계
    웹 브라우저는 HTML 파일을 읽고 렌더링(화면에 출력) 함
    HTML은 구조, CSS는 디자인, JavaScript는 동작 담당
    브라우저는 DOM(Document Object Model)으로 HTML 해석
